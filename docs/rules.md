# 개발 및 AI 에이전트 행동 수칙 (Rules)

1. **언어 원칙**
   - 시스템의 최종 답변, 에이전트의 진행 상황 보고, 코드의 주요 주석은 모두 **한국어**로 작성한다.

2. **구현 원칙**
   - **투명성 (Transparency)**: AI의 추론 과정(SPARQL 쿼리, 근거 데이터)을 사용자에게 숨기지 않고 시각화한다.
   - **모듈화**: `generate_sparql`, `execute_sparql`, `generate_answer` 등 핵심 로직은 독립적인 함수/모듈로 분리하여 유지보수성을 높인다.
   - **오류 처리**: SPARQL 쿼리 실행 실패 시, 원인을 파악할 수 있는 에러 메시지를 제공하거나 재시도 로직(Optional)을 고려한다.

3. **보고 체계**
   - 각 Step(마일스톤)이 완료될 때마다 사용자에게 다음 내용을 포함하여 **한글**로 브리핑한다.
     - 완료된 작업 내역
     - 주요 구현 사항 및 이슈
     - 다음 단계 계획

4. **보안**
   - API Key 등 민감 정보는 절대 소스 코드에 하드코딩하지 않고 `.env` 파일을 통해 관리한다.
